Building Depot v3.1¶

Overview¶

This is the official documentaion of BuildingDepot v3.1. BuildingDepot is essentialy an Extensible and Distributed Architecture for Sensor Data Storage, Access and Sharing.

Building Depot has two essential components a CentralService and a Data Service :

CentralService - The central metadata repository for Building Depot. Information regarding buildings, users, sensors, etc. is stored here.

DataService - The repository that stores sensor data. Exposes a RESTful API for retrieving data and inserting data into the system.

An institution has a single CentralService that houses data of all buildings and user accounts on campus, and multiple DataServices, each of which may contain sensor data of several buildings.

A DataService may belong to any single administrative group that requires sole control over who can access the underlying sensor data points. Different buildings on a campus might desire their own DataService.
